# Sistema_Registro_Usuario
#Sistema de Registro de Usuarios

Proyecto académico que implementa la documentación, pruebas y mantenimiento para un sistema básico de registro de usuarios.

Descripción del Caso

El sistema permite registrar usuarios mediante un formulario que recibe nombre, correo y contraseña. Los datos se validan y se almacenan de forma segura.

 Objetivos del Proyecto

Documentar requerimientos funcionales y no funcionales.

Elaborar un plan de pruebas funcionales.

Definir y proponer un tipo de mantenimiento.

Demostrar el uso de control de versiones en GitHub.

Utilizar Markdown para documentación técnica.

 Requerimientos 

Registrar usuario con nombre, correo y contraseña.

Validar correos duplicados.

Cifrar contraseñas.

Permitir inicio de sesión.

 Tabla de Pruebas Funcionales
Caso	Descripción	Resultado Esperado
1	Registro exitoso	Usuario creado
2	Correo duplicado	Mensaje de error
3	Inicio de sesión	Acceso permitido

 Tipo de Mantenimiento Propuesto

Perfectivo
Implementación de mejoras en seguridad, rendimiento y experiencia del usuario.

 Investigación Markdown

Incluye conceptos, ejemplos y ventajas de su uso en proyectos.

 Reflexión sobre el control de versiones

El uso de Git y GitHub permite organizar mejor el proyecto, mantener historial de cambios, revertir errores, trabajar colaborativamente y documentar la evolución del sistema. Es una herramienta fundamental en el desarrollo de software profesional.
