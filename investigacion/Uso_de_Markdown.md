Investigación: Uso de Markdown en Proyectos de Software
¿Qué es Markdown?

Markdown es un lenguaje de marcado ligero que permite dar formato al texto de forma simple y legible. Se usa ampliamente en documentación técnica.

¿Por qué se utiliza en proyectos de software?

Es fácil de escribir y leer.

Compatible con GitHub, GitLab, GitBook y VSCode.

Ideal para documentación de repositorios.

Permite versionar documentación igual que el código.

Ejemplo de uso:
Encabezados
# Título
## Subtítulo

Listas
- Elemento 1
- Elemento 2

Tabla
| Campo | Descripción |
|-------|-------------|
| Nombre | Nombre del usuario |


Ventajas de usar Markdown con GitHub

Visualización automática.

Historial de versiones.

Fácil colaboración.

Compatible con issues y wikis.
